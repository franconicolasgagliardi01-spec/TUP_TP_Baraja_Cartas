## Descripción General
El objetivo de este trabajo práctico es diseñar e implementar una aplicación orientada a objetos en Java que modele una Baraja Española de 40 cartas (Espadas, Bastos, Oros y Copas; números del 1 al 12, excluyendo 8 y 9). 

El sistema utiliza colecciones dinámicas (`ArrayList`) para gestionar las cartas y aplica conceptos fundamentales de POO (abstracción, encapsulamiento y distribución de responsabilidades), preparando la arquitectura para futuras expansiones como interfaces gráficas o lógica de juegos complejos.

## Funcionalidades Requeridas
* Barajar la baraja.
* Obtener la siguiente carta disponible.
* Consultar la cantidad de cartas disponibles.
* Repartir una cantidad determinada de cartas.
* Mostrar las cartas que ya han sido repartidas.
* Mostrar las cartas que permanecen disponibles en la baraja.
